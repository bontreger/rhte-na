HA Deployment Lab

In this lab’s scenario, you are a consultant assigned to MitziCom, a telecommunications company. MitziCom provides hosting and cloud services to a variety of clients, ranging from medium-sized companies to enterprise giants.

MitziCom has asked you to lead a 30- to 40-hour proof-of-concept (POC) using Red Hat OpenShift Container Platform. The purpose of the POC is to determine the feasibility of using Red Hat OpenShift Container Platform as a target for internal and client workloads.

The lab details a recommended process for meeting MitziCom’s requirements, as defined in the lab’s goals. Information about the infrastructure, DNS names, passwords, and more is provided throughout the lab.

Tip
 Because the lab does not include all of the required steps and commands, you may want to consult the OpenShift Container Platform product documentation. This link is especially useful for research and troubleshooting. Your instructor is also available to help you.
Goals

In this lab, you deploy and configure OpenShift 3 on a group of servers to meet these requirements:

  • Configure Red Hat Enterprise Linux 7.3 hosts for OpenShift deployment

  • Deploy a highly available OpenShift cluster

  • Configure the OpenShift cluster

Provisioned Environment Hosts

  • Administration server: bastion.$GUID.example.opentlc.com, bastion.$GUID.internal

  • NFS server: support1.$GUID.example.opentlc.com, support1.$GUID.internal

  • IPA Server: ipa.shared.example.opentlc.com (shared resource for all students)

  • Load balancer: loadbalancer.$GUID.example.opentlc.com, loadbalancer1.$GUID.internal

  • 3 OpenShift master nodes: master{1,2,3}.$GUID.example.opentlc.com, master{1,2,3}.$GUID.internal

  • 2 OpenShift infrastructure nodes: infranode{1,2}.$GUID.example.opentlc.com, infranode{1,2}.$GUID.internal

  • 2 OpenShift worker nodes: node{1-2}.$GUID.example.opentlc.com, node{1-2}.$GUID.internal

3. Deploy Highly Available OpenShift Cluster

In this section, you use the Ansible advanced installer to deploy OpenShift as a clustered, highly available installation that includes a load balancer in front of the API servers. The environment includes three masters, two infra nodes, two worker nodes, and the load balancer.

The /etc/ansible/hosts Ansible hosts file is used to define and configure the OpenShift cluster.

Upon successful execution of the ansible_playbook command, an HA OpenShift cluster is deployed.

Expect the OpenShift cluster to have the following characteristics:

  • Three master hosts in the deployment

  • A load balancer to access the masters

  • Two infra nodes, each running a router container

  • An integrated registry pod backed by a persistent volume (PV) storage (NFS)

  • Router pods deployed, configured, and running on each infranode in the cluster

  • Aggregated logging configured and working

  • Metrics collection configured and working

  • Worker nodes labeled as env=app and infra nodes labeled as env=infra

  • The env=infra node selector defined in the default namespace

  • A *.apps.${GUID}.example.opentlc.com wildcard DNS entry that points to the infra nodes

  • The load balancer configured with loadbalancer.${GUID}.example.opentlc.com as the external DNS entry

  • Custom "Intermediary CA" key and certificate deployed on the masters

3.1. Prepare Ansible for OpenShift Deployment

  1. On the bastion host, install the atomic-openshift-utils package, which includes the installer and has Ansible and the playbooks as dependencies:

    # yum -y install atomic-openshift-utils atomic-openshift-clients

  2. If you have an IdM server in the environment, download the CA certificate by running the following script on your bastion system:

    # wget http://ipa.shared.example.opentlc.com/ipa/config/ca.crt -O /root/ipa-ca.crt
    Important
    		 You need this certificate authenticated with the ipa.shared.example.opentlc.com IDM server. Ansible distributes the CA certificate from the IDM server to the master nodes so that OpenShift can make SSL-enabled LDAP connections to the IDM server.

  3. Get the "Intermediary CA" key and certificate to be used in the signing of all keys in the OpenShift system.

    # wget http://admin.shared.example.opentlc.com/certs/ca.crt -O /root/intermediary_ca.crt
# wget http://admin.shared.example.opentlc.com/certs/ca.key -O /root/intermediary_ca.key

5. Configure OpenShift Cluster

In this section, you create persistent volumes (PVs) for users to consume, and you verify LDAP authentication.

  • You create 25 PVs with these parameters:

    • Size: 5 GB

    • PV access: ReadWriteOnce

    • ReclaimPolicy: Recycle

  • You create 25 PVs with these parameters:

    • Size: 10 GB

    • PV access: ReadWriteMany

    • ReclaimPolicy: Retain

5.1. Create Different PVs for Users

  1. Create directories on the support1.$GUID.internal NFS host to be used as PVs in the OpenShift cluster:

    ssh support1.$GUID.internal
sudo bash
mkdir -p /srv/nfs/user-vols/pv{1..200}

for pvnum in {1..50} ; do
echo /srv/nfs/user-vols/pv${pvnum} *(rw,root_squash) >> /etc/exports.d/openshift-uservols.exports
chown -R nfsnobody.nfsnobody  /srv/nfs
chmod -R 777 /srv/nfs
done

systemctl restart nfs-server

  2. Create 25 definition files for PVs pv1 to pv25 with a size of 5 GB:

    ssh master1.$GUID.internal
sudo bash
export GUID=`hostname|awk -F. '{print $2}'`
echo $GUID

export volsize="5Gi"
mkdir /root/pvs
for volume in pv{1..25} ; do
cat << EOF > /root/pvs/${volume}
{
  "apiVersion": "v1",
  "kind": "PersistentVolume",
  "metadata": {
    "name": "${volume}"
  },
  "spec": {
    "capacity": {
        "storage": "${volsize}"
    },
    "accessModes": [ "ReadWriteOnce" ],
    "nfs": {
        "path": "/srv/nfs/user-vols/${volume}",
        "server": "support1.${GUID}.internal"
    },
    "persistentVolumeReclaimPolicy": "Recycle"
  }
}
EOF
echo "Created def file for ${volume}";
done;

  3. Create 25 definition files for PVs pv26 to pv50 with a size of 10 GB:

    export volsize="10Gi"
for volume in pv{26..50} ; do
cat << EOF > /root/pvs/${volume}
{
  "apiVersion": "v1",
  "kind": "PersistentVolume",
  "metadata": {
    "name": "${volume}"
  },
  "spec": {
    "capacity": {
        "storage": "${volsize}"
    },
    "accessModes": [ "ReadWriteMany" ],
    "nfs": {
        "path": "/srv/nfs/user-vols/${volume}",
        "server": "support1.${GUID}.internal"
    },
    "persistentVolumeReclaimPolicy": "Retain"
  }
}
EOF
echo "Created def file for ${volume}";
done;

  4. Use oc create to create all of the PVs you defined:

    cat pvs/* | oc create -f -
