Lab #3: Using KIE server API with an Angular UI

Task #1 - Modify Angular UI

  1. Make sure to stop and delete any running KIE containers.  JBoss BPMS should be running.
  2. Git clone the BPMSTechExchangeRepoUI into business-central https://jhurlock@gitlab.consulting.redhat.com/americas-tech-exchange-2017/processapps-60/BPMSTechExchangeRepo.git
  3. Git clone the BPMSAngularWebAppUI project into a local directory https://jhurlock@gitlab.consulting.redhat.com/americas-tech-exchange-2017/processapps-60/BPMSAngularWebAppUI.git
  4. In the BPMSTechExchangeRepoUI project take a look at the InterviewProcess, the WorkExperience rule, and the Candidate Data Object to get an idea of how the process flows.
  5. Build and deploy the BPMSTechExchangeRepoUI project through Business Central.
  6. Create a KIE container and name it TechExchangeOrg2017:InterviewProj:1.0  Set the runtime strategy in the container Process Configuration to Per Process Instance.  Start the container.
  7. Go to the BPMSAngularWebAppUI that you cloned in step 3.  Run mvn clean install.
  8. After the BPMSAngularWebAppUI is successfully deployed to EAP open a browser and go to http://localhost:8080/AngularDynProj
  9. Use jboss for the user and bpms as the password.  Click Login.
  10. After logging in navigate to Processes - InterviewProcess.  NOTE:  If the InterviewProcess does not display then refresh or stop and start the KIE container.
  11. Notice how the Package and Version of the process are blank.  In the next few steps you’ll add them in and redeploy the BPMSAngularWebAppUI.
  12. Go to BPMSAngularWebAppUI/src/app/
  13. The file that you will need to edit to add in the Package and Version is Process.component.html.  Take a look at how the Container ID is being set.  That should give you an idea of how to set the Package and Version.  The processDefinitions variable is defined in Process.component.ts. The REST request is also defined in Process.component.ts.  For the full KIE server remote API go to http://localhost:8080/kie-server/docs/
  14. Once you have added in the appropriate JSON elements for the Pacakge and Version rebuild the BPMSAngularWebAppUI.  mvn clean install
  15. Once the BPMSAngularWebAppUI project is successfully deployed reload http://localhost:8080/AngularDynProj and go to Processes - InterviewProcess.  You should now see values for Package and Version.
  16. Click on the Candidate button and fill out firstName, lastName, and enter a value > 5 in for yearsOfExperience.  Click Start Process.
  17. Go to Tasks - Open Tasks.  You should see an open task for Interviewer 1.
  18. Claim the task and notice that the owner will update to jboss.  Start the task.
  19. Open the Task Forms link and click the Candidate button.  Change the passInterview1 to true.  Click Complete.
     
  20. The page should refresh and an Interview 2 task should now exist.  Claim, Start, and Complete the task.  This concludes Task #1 of this lab.

Task #2 - Create a new process and regenerate Angular UI

  1. In Business Central create a new Data Object named Expenses in the same package the Candidate data object is in.
  2. Add the following fields and Save the data object.

Identifier

Type

firstName

String

lastName

String

expenseAmount

BigDecimal

approved

boolean

  1. Create a new process in the same package the InterviewProcess is in.  Name the process ExpenseApprovalProcess.
  2. Add a Start event --> User Task --> End event
  3. Import the Expense data object into the ExpenseApprovalProcess
  4. Add a process variable named Expenses that is mapped to the Expenses data object
  5. On the User Task add Data Inputs and Outputs.   Put Name, Data Type, and Source as Expenses.
  6. Add jboss, busadmin in the Actors section of the User Task.  Save the process.
  7. Undeploy the InterviewProj process deployment under Deploy - Process Deployments.
  8. Open the project editor and build and deploy InterviewProj.
  9. Stop and Start your KIE container (TechExchangeOrg2017:InterviewProj:1.0)
  10. Download the latest InterviewProj-1.0.jar from Authoring - Artifact Repository
  11. Git clone the BPMSAngularGeneratorUI project into a local directory. https://jhurlock@gitlab.consulting.redhat.com/americas-tech-exchange-2017/processapps-60/BPMSAngularGeneratorUI.git
  12. Go to the BPMSAngularGeneratorUI project and run the below command with appropriate values:
    mvn exec:java -Dexec.mainClass=com.redhat.GenerateUI -Dexec.args="TechExchangeOrg2017:InterviewProj:1.0 '<PATH TO BPMSAngularWebApp>' '<PATH TO InterviewProj-1.0.jar>'"
  13. Make sure EAP is still running and go to the BPMSAngularWebApp and run:
    mvn clean install
  14. Open a browser and go to http://localhost:8080/BPMSAngularWebApp
  15. You should now see the ExpenseApprovalProcess listed under the Processes drop down.
  16. Select the ExpenseApprovalProcess.  You should see the new form that was generated to start the process instance.
  17. Fill in fristName, lastName, and expenseAmount and click Start Process.
  18. Click on Tasks - Open Tasks and you should see a new task created.  Claim, Start, and Complete the task.  This concludes Task #2 of this lab.
